# PRD — AI 트레이딩 카드 / 리더보드 이벤트 앱

> 단일 진실 원천: [`/CLAUDE_PROJECT_BRIEF.md`](../CLAUDE_PROJECT_BRIEF.md)
> 이 문서는 브리핑을 제품 요구사항으로 구체화한다. 충돌 시 브리핑이 우선한다.

## 1. 한 줄 정의

> 내 사진이 AI 카드가 되고, AI가 자동으로 점수·등급을 매겨 리더보드에서 경쟁하는 **공유형 트레이딩 카드 앱**.

"포켓몬 카드 만들기 앱"이 **아니다.** 사용자가 점수/능력치를 직접 입력하는 카드 편집 툴도 아니다. 핵심은 **AI 자동 채점 + 리더보드 경쟁 + SNS 바이럴**이다.

## 2. 차별점 (무엇을 하고, 하지 않는가)

| 한다 (O) | 하지 않는다 (X) |
|---|---|
| AI가 사진 퀄리티·카드 적합도 분석 | 단순 프레임에 사진 넣기 |
| 점수/등급/능력치 **자동** 생성 | 사용자가 점수/능력치 **수동 수정** |
| 서버 기준 점수·리더보드 | 클라이언트 점수 신뢰 |
| 리더보드 경쟁 + 이벤트 상품 | 결제로 순위 상승 |
| 공유 → 앱 유입 바이럴 루프 | 단순 이미지 저장만 |

## 3. 핵심 제약 (절대 원칙)

1. **점수/등급/능력치는 사용자가 수정 불가** — 서버 계산값만 사용.
2. **점수 계산·리더보드는 서버 기준.** 클라이언트 값 신뢰 금지.
3. **상품 랭킹은 점수 기준**, 공유 수는 바이럴 분석용으로만.
4. **이벤트 수상자는 운영자 검수 후 확정.**
5. **포켓몬 IP 직접 사용 금지** — 명칭/로고/GX·EX·VMAX·VSTAR/에너지 아이콘/공식 레이아웃 복제 금지. 장르 문법만 차용. 세부는 [`IP_RISK_GUIDE.md`](./IP_RISK_GUIDE.md).
6. **UGC 앱** — 신고/차단/관리자 숨김/랭킹 제외 필수.

## 4. 사용자가 조정 가능한 것 (허용)

다시 촬영 · 다른 사진 재분석 · 카드 프레임 재생성 · 공유 이미지 스타일 선택 · 닉네임 · 공개/비공개 · 리더보드 제출 여부 · 이벤트 참가 동의.

## 5. 페르소나

- **참가자(주력)**: 사진 찍어 등급 받고 자랑/공유하려는 10~30대. "내 사진 SSS 떴다"가 동기.
- **공유 유입자**: 카카오/인스타 링크 보고 들어와 "나도 만들기"로 전환되는 신규.
- **운영자(Admin)**: 부적절·부정 카드 검수, 이벤트/상품 관리.

## 6. 최종 사용자 플로우 (웹 우선)

```text
1. 진입(앱/공유 링크)
2. 사진 촬영(1컷) 또는 앨범 업로드
3. 카드 영역 가이드에 맞춰 크롭/정렬
4. 서버 업로드
5. AI 이미지 분석 (선명도·노출·해상도·구도·피사체·색감)
6. 퀄리티 점수(100점) 계산  ← 서버
7. 등급/희귀도/속성/PWR/스킬·패시브 자동 생성  ← 서버
8. 카드 이미지 자동 렌더링 (+ 공유용 5종)
9. 리더보드 등록
10. 카드 상세 랜딩(/card/{id}) 생성
11. 카카오/인스타/페이스북 공유
12. 공유 유입자 → "나도 만들기" 전환
```

> 촬영은 **1컷**(브리핑 정정사항). 4컷 아님. 촬영 UX 구조만 포토부스 참고작에서 차용.

## 7. 채점·등급 요약 (상세는 TECH_SPEC)

- 점수 100점 = 선명도25 + 노출15 + 해상도15 + 구도20 + 피사체15 + 색감10 − 패널티.
- 등급/희귀도 매핑:

| 점수 | 등급 | 희귀도 |
|---:|---|---|
| 97–100 | SSS+ | Mythic |
| 93–96 | SSS | Legendary |
| 88–92 | SS | Ultra Rare |
| 80–87 | S | Super Rare |
| 70–79 | A | Rare |
| 55–69 | B | Uncommon |
| 0–54 | C | Common |

- 능력치: `PWR = 60 + 품질점수×2` (60~260), 스킬파워 = `품질×0.8 + 피사체보너스(10~40)` 10단위 반올림.
- 속성(자체 명칭): Aqua/Flame/Terra/Storm/Neon/Shadow/Metal/Spirit/Solar/Lunar.

## 8. 기능 범위 (MVP 1.0 / 1.5 / 2.0)

상세·구현순서는 [`MVP_ROADMAP.md`](./MVP_ROADMAP.md). 요약:

- **1.0**: 익명/소셜 식별, 촬영·업로드, 자동 크롭, 서버 업로드, 기본 품질 분석(mock/OpenCV), 자동 점수·등급·능력치, 자체 프레임 5~10종, 카드 렌더링, 일간/주간 리더보드 + 주간 이벤트, 카카오·인스타·페북 공유, 이미지 저장, 카드 상세 웹페이지, 신고, 관리자 숨김/삭제/랭킹 제외.
- **1.5**: 피사체 인식 고도화, 속성 추천 고도화, 스킬/패시브명 생성 고도화, 공유 유입 분석, 시즌 이벤트, 고화질 내보내기, 워터마크 제거 유료화 검토, TOP10 검수 플로우.
- **2.0**: 사진→일러스트 변환, 홀로/포일, 3D 미리보기, 컬렉션 북, 친구 초대, 커뮤니티, 브랜드 제휴, 배틀 모드, 프리미엄 프레임.

## 9. 성공 기준 (MVP)

- 업로드 후 **30초 이내** 카드 결과 표시.
- 점수/등급/희귀도/속성/PWR/스킬 자동 입력, 사용자 수정 불가.
- 카드가 리더보드에 등록되고 본인 순위 확인 가능.
- 카카오/인스타/페북 공유 가능, 공유 링크 클릭 시 카드 상세 페이지 오픈, 신규 유저 "나도 만들기" 진입.
- 관리자가 부적절·부정 카드 숨김/삭제/랭킹 제외 가능.
- **포켓몬 카드와 혼동될 UI/명칭/아이콘/로고 미사용.**

## 10. 수익화 (초기 바이럴 우선)

프리미엄 프레임 / 고화질 저장 / 워터마크 제거 / 시즌 한정 프레임 / 브랜드 콜라보 / 광고 시청 후 추가 분석 / 구독.
**금지**: 결제로 점수·순위 상승, 상품 이벤트에서 결제 유저 우대, 이벤트 참가권 유료 판매.
