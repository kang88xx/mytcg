# MVP_ROADMAP — 구현 로드맵

> 스택: Next.js 풀스택 웹 우선. 점수/랭킹 서버 권위. 구현 전 본 로드맵 승인 필요.

## 단계 개요

```text
P0 스캐폴딩 → P1 카드 생성 플로우 → P2 채점/렌더 → P3 리더보드
→ P4 공유/랜딩 → P5 신고/관리자 → P6 이벤트/하드닝 → 베타
```

## P0 — 프로젝트 스캐폴딩
- Next.js(App Router, TS) + Prisma + Postgres + Redis + S3 클라이언트 + BullMQ 큐 세팅.
- 환경변수, `ImageScorer` 인터페이스 골격, 시드 데이터.
- **DoD**: `npm run dev`로 빈 앱 + DB 마이그레이션 통과.

## P1 — 카드 생성 플로우 (1컷)
- 촬영(웹캠, 카드 가이드라인)/앨범 업로드, 크롭·회전, 최소 해상도/저조도/흔들림 경고.
- `POST /api/cards` → S3 원본 저장 → `processing` 반환 → 큐 적재.
- **DoD**: 사진 1컷 업로드 시 `cardId`+`processing` 수신, 원본 S3 저장.

## P2 — 채점 + 카드 렌더 (서버)
- `HeuristicScorer`(sharp/OpenCV 통계): 선명도/노출/해상도/구도/피사체/색감 → 100점.
- 등급/희귀도/속성/PWR/스킬·패시브 자동 생성(수정 불가). `imageHash`/`pHash` + 중복·패널티.
- 서버 카드 렌더(`@vercel/og`) → `card_original` + 공유 5종 → CDN.
- 결과 폴링/SSE로 `active` 전환, 결과 화면 표시(수정 UI 없음).
- **DoD**: 업로드→**30초 내** 점수/등급/속성/PWR/스킬 자동 채워진 카드 이미지 표시.

## P3 — 리더보드
- 등록 시 Redis ZSET 반영(daily/weekly/monthly/all_time/event).
- `GET /api/leaderboards` + 내 순위, 탭 UI, 계정당 이벤트 대표 1개 규칙.
- **DoD**: 생성 카드가 리더보드 노출, 본인 순위 확인.

## P4 — 공유 / 카드 상세 랜딩
- `/card/{id}` 서버 렌더 + 동적 OG 메타.
- 카카오(JS SDK)/인스타(스토리·피드 버튼 분리, 앱 이동)/페북(Share Dialog) + 이미지 저장.
- `POST /api/share-logs` 로깅. "나도 만들기" 전환 CTA.
- **DoD**: 공유 링크 클릭→상세 페이지 미리보기 정상, 신규 유저 생성 진입.

## P5 — 신고 / 관리자
- `POST /api/cards/{id}/report`, 신고 임계 자동 `reported`.
- `/admin`: 카드/신고 목록, 숨김/삭제/랭킹 제외, 사용자 제한, 중복 검토.
- 부적절 이미지 기본 필터(MVP 휴리스틱/검수 큐).
- **DoD**: 관리자가 카드 숨김/삭제/랭킹 제외 가능, 신고 동작.

## P6 — 이벤트 / 하드닝
- 주간 이벤트 랭킹 + 약관(`rulesUrl`, 플랫폼 비후원 고지), TOP10 검수 확정 플로우.
- Rate limit(하루 3회), 이상 제출 탐지, 개인정보 처리방침/카드 삭제 요청.
- **DoD**: 주간 이벤트 운영 가능, 제출 제한·검수 동작.

---

## 버전 범위 (브리핑 §15)

**MVP 1.0** = P1~P6 전부 (필수 기능 일체).
**MVP 1.5**: 피사체 인식 고도화, 속성/스킬·패시브명 고도화, 공유 유입 분석, 시즌 이벤트, 고화질 내보내기, 워터마크 제거 유료화 검토, TOP10 검수 강화.
**MVP 2.0**: 사진→일러스트 변환, 홀로/포일, 3D 미리보기, 컬렉션 북, 친구 초대, 커뮤니티, 브랜드 제휴, 배틀 모드, 프리미엄 프레임.

## MVP 성공 기준 (게이트)

```text
□ 업로드 후 30초 내 카드 생성
□ 점수/등급/희귀도/속성/PWR/스킬 자동, 사용자 수정 불가
□ 리더보드 등록 + 본인 순위 확인
□ 카카오/인스타/페북 공유, 링크→상세→나도 만들기
□ 관리자 숨김/삭제/랭킹 제외
□ 포켓몬 혼동 UI/명칭/아이콘/로고 미사용 (IP 체크리스트 통과)
```

## 권장 구현 우선순위 (브리핑 §19.3)

1. 카드 생성 플로우 → 2. 이미지 업로드 → 3. 서버 점수(mock/OpenCV) → 4. 카드 렌더 → 5. 리더보드 → 6. 공유 랜딩 → 7. 공유 버튼 → 8. 신고 → 9. 관리자 기본.
AI 분석은 mock/OpenCV로 시작, `ImageScorer` 인터페이스로 ML 교체 가능하게 유지.
