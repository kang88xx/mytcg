# 프로젝트 작업 지시서: AI 트레이딩 카드 / 리더보드 이벤트 앱

## 0. 프로젝트 개요

사용자가 사진을 촬영하거나 업로드하면, 앱이 이미지를 자동 분석하여 카드 형태의 이미지를 생성한다.

단순한 “사진 넣는 카드 만들기 앱”이 아니라, 핵심은 다음이다.

- 사용자가 직접 사진을 찍거나 업로드한다.
- AI가 이미지 퀄리티를 분석한다.
- 분석 결과를 바탕으로 점수, 등급, 희귀도, 능력치, 속성, 스킬명 등을 자동 생성한다.
- 사용자는 능력치나 점수를 수동으로 수정할 수 없다.
- 생성된 카드는 리더보드에 등록된다.
- 주간/월간 랭킹 기반 이벤트를 운영하며, 상위 순위자에게 상품을 지급할 수 있다.
- 생성된 카드는 카카오톡, 인스타그램, 페이스북 등으로 공유할 수 있어야 한다.
- 공유는 단순 이미지 저장이 아니라, 앱 유입과 리더보드 참여로 이어지는 바이럴 구조여야 한다.

기본 참고 이미지는 포켓몬 카드 형태의 세로형 트레이딩 카드다. 다만 실제 구현에서는 포켓몬 IP, 로고, 고유 UI, 고유 명칭, 카드 레이아웃을 직접 복제하지 않고, “트레이딩 카드 장르의 익숙함”만 활용한 독자 디자인으로 구현해야 한다.

---

## 1. 핵심 포지셔닝

이 앱은 “포켓몬 카드 만들기 앱”이 아니다.

정확한 포지셔닝은 다음이다.

> 내 사진이 AI 카드가 되고, AI가 자동으로 점수와 등급을 매겨 리더보드에서 경쟁하는 공유형 트레이딩 카드 앱

핵심 차별점은 다음이다.

- 단순 카드 프레임에 사진 넣기 X
- 사용자가 점수/능력치를 직접 수정하는 카드 만들기 X
- AI가 사진 퀄리티와 카드 적합도를 분석 O
- 자동 점수/등급/능력치 생성 O
- 리더보드 경쟁 O
- SNS 공유를 통한 바이럴 O
- 이벤트 상품 지급 O

## 2. 기존 5단계 수정사항

기존 구상에는 “자동 생성된 내용을 사용자가 수정 가능” 단계가 있었으나, 이 기능은 제거한다.

### 제거해야 하는 기능

- 사용자가 점수 수동 수정
- 사용자가 등급 수동 수정
- 사용자가 HP/공격력/능력치 수동 수정
- 사용자가 리더보드에 반영되는 수치를 임의 변경
- 사용자가 AI 판정 결과를 조작할 수 있는 기능

### 허용 가능한 기능

- 다시 촬영하기
- 다른 사진으로 다시 분석하기
- 카드 프레임 재생성
- 공유용 이미지 스타일 선택
- 닉네임 설정
- 공개/비공개 여부 선택
- 리더보드 제출 여부 선택
- 상품 이벤트 참가 동의

리더보드 신뢰성을 위해 점수와 등급은 서버 기준으로 계산하고, 사용자가 직접 변경할 수 없어야 한다.

## 3. 최종 사용자 플로우

앱의 기본 플로우는 다음과 같다.

```text
1. 앱 실행
2. 사진 촬영 또는 앨범에서 이미지 업로드
3. 이미지 크롭/정렬
4. 서버에 이미지 업로드
5. AI 이미지 분석
6. 이미지 퀄리티 점수 계산
7. 카드 등급/희귀도/속성/능력치/스킬명 자동 생성
8. 카드 이미지 자동 렌더링
9. 리더보드 등록
10. 카드 상세 페이지 생성
11. 카카오톡/인스타그램/페이스북 공유
12. 공유 링크 유입자 앱 설치 또는 웹에서 카드 생성 유도
```

## 4. 주요 기능 목록

### 4.1 사진 촬영/업로드

필수 기능:

- 카메라 촬영
- 앨범 이미지 업로드
- 카드 이미지 영역에 맞춘 크롭
- 확대/축소/회전
- 기본 자동 크롭
- 최소 해상도 체크
- 너무 작은 이미지 업로드 제한
- 너무 어두운 이미지 경고
- 흔들린 이미지 경고

중요:

- 촬영 화면에는 카드 이미지 박스에 맞는 가이드라인을 제공한다.
- 사용자가 사진을 넣는 영역은 카드 중앙의 메인 이미지 박스다.
- 이미지 분석용 원본은 서버에 업로드하되, 개인정보 보호 정책에 명시해야 한다.

---

### 4.2 AI 이미지 분석

이미지 분석 항목은 다음과 같다.

- 선명도
- 초점 상태
- 흔들림 여부
- 밝기/노출
- 노이즈
- 해상도
- 구도
- 피사체 중앙 정렬 여부
- 피사체가 잘렸는지 여부
- 얼굴/동물/사물/풍경 등 피사체 카테고리
- 주요 색상
- 분위기
- 카드화 적합도
- 중복 이미지 여부
- 부적절 이미지 여부

주의:

- 사람 얼굴이 포함될 수 있으나, 개인 신원 식별 기능은 구현하지 않는다.
- 외모 점수로 보이면 안 된다.
- 점수명은 반드시 “카드 완성도 점수”, “이미지 퀄리티 점수”, “AI 카드 점수”, “배틀 카드 점수” 같은 중립적 표현을 사용한다.
- “외모 점수”, “얼굴 점수”, “잘생김 점수”, “예쁨 점수” 등은 사용하지 않는다.

---

### 4.3 점수 산정 방식

최종 점수는 100점 만점으로 계산한다.

예시 산식:

```text
최종 점수 =
선명도 25%
노출/밝기 15%
해상도 15%
구도 20%
피사체 인식도 15%
색감/카드화 적합도 10%
- 중복/부적절/저품질 패널티
```

각 항목 설명:

| 항목 | 비중 | 설명 |
|---|---:|---|
| 선명도 | 25% | 흐림, 초점, 흔들림 여부 |
| 노출/밝기 | 15% | 너무 어둡거나 밝은 사진 감점 |
| 해상도 | 15% | 카드 이미지로 렌더링했을 때 깨지는지 |
| 구도 | 20% | 피사체가 중앙에 있고 잘리지 않았는지 |
| 피사체 인식도 | 15% | 사람, 동물, 사물, 풍경 등 인식 신뢰도 |
| 색감/카드화 적합도 | 10% | 카드 이미지로 보기에 좋은 색감인지 |

패널티 예시:

```text
동일 이미지 재업로드: -30점 또는 랭킹 제외
심한 저해상도: -20점
너무 어두움: -10점
피사체 없음: -15점
부적절 이미지: 점수 생성 중단 및 제출 불가
워터마크/스크린샷 의심: -10점 또는 검수 대상
```

---

### 4.4 등급 매핑

점수에 따라 자동 등급과 희귀도를 부여한다.

| 점수 | 등급 | 희귀도 |
|---:|---|---|
| 97~100 | SSS+ | Mythic |
| 93~96 | SSS | Legendary |
| 88~92 | SS | Ultra Rare |
| 80~87 | S | Super Rare |
| 70~79 | A | Rare |
| 55~69 | B | Uncommon |
| 0~54 | C | Common |

등급명은 추후 브랜드 톤에 맞춰 변경 가능하다.

---

### 4.5 능력치 자동 생성

사용자가 직접 수정할 수 없는 자동 능력치를 생성한다.

예시 능력치:

```text
PWR
AURA
FOCUS
STYLE
LUCK
RARITY
```

포켓몬 카드의 “HP”와 너무 동일하게 보이지 않도록, HP 대신 다음 명칭을 고려한다.

- PWR
- SCORE
- AURA
- CP
- POWER

예시 계산식:

```text
PWR = 60 + 품질점수 × 2
최소 60, 최대 260
```

공격력 또는 스킬 수치 예시:

```text
기본 스킬 파워 = 품질점수 × 0.8
피사체 인식 보너스 = +10~40
최종 스킬 파워 = 10 단위 반올림
```

예시 결과:

```text
점수: 96.4
등급: SSS
희귀도: Legendary
속성: Aqua
PWR: 250
대표 스킬: Blue Impact 100+
패시브: Crystal Focus
```

---

### 4.6 속성 자동 추천

사진의 주요 색상, 피사체, 분위기를 기반으로 속성을 자동 추천한다.

예시 속성:

```text
Aqua
Flame
Terra
Storm
Neon
Shadow
Metal
Spirit
Solar
Lunar
```

주의:

- 포켓몬 에너지 아이콘과 유사한 아이콘을 만들지 말 것.
- 원형 안에 단순 심볼만 넣는 방식은 포켓몬 에너지 카드와 유사해질 수 있으므로 피한다.
- 자체 문양과 자체 시각 언어를 사용한다.

속성 아이콘 예시:

```text
Aqua = 물방울 + 파형 문양
Flame = 삼각 불꽃 + 균열 텍스처
Terra = 육각 결정
Storm = 번개가 아니라 회오리 전류
Neon = 빛나는 렌즈
Shadow = 초승달 그림자
Metal = 금속 결정
Spirit = 오라 링
Solar = 태양 파편
Lunar = 달 조각
```

## 5. 리더보드 기능

리더보드는 앱의 핵심 기능이다.

### 5.1 리더보드 종류

필수:

```text
일간 랭킹
주간 이벤트 랭킹
월간 랭킹
전체 명예의 전당
```

추가 가능:

```text
카테고리별 랭킹
친구 초대 랭킹
공유 유입 랭킹
신규 유저 랭킹
시즌별 이벤트 랭킹
```

추천 리더보드 구조:

| 구분 | 내용 |
|---|---|
| 일간 랭킹 | 오늘 생성된 카드 TOP 100 |
| 주간 이벤트 랭킹 | 상품 지급 대상 랭킹 |
| 월간 명예의 전당 | 누적 최고 점수 |
| 카테고리 랭킹 | 인물, 반려동물, 음식, 풍경, 자유 |
| 공유 랭킹 | 공유 링크 유입/조회 기반, 단 상품 랭킹과 분리 권장 |

---

### 5.2 리더보드 제출 기준

- 서버에서 점수가 계산된 카드만 제출 가능
- 부적절 이미지 제외
- 중복 이미지 제외
- 이벤트 기간 내 생성된 카드만 해당 이벤트 랭킹 반영
- 계정당 이벤트 대표 카드 1개만 상품 랭킹 반영
- 동일 사용자의 여러 카드 중 최고점 1개만 표시 가능
- TOP 10 또는 상품 대상자는 운영자 검수 후 최종 확정

---

필수 구현:


## 6. 공유 기능 설계

이 앱은 저장 기능보다 공유 기능이 중요하다.

단순히 이미지만 저장하는 것이 아니라, 공유가 앱 유입과 리더보드 참여로 연결되어야 한다.

### 6.1 공유 구조

생성물은 두 가지로 관리한다.

```text
1. 카드 이미지 파일
2. 카드 상세 랜딩 페이지
```

카드 상세 URL 예시:

```text
https://service-domain.com/card/{cardId}
```

공유 링크를 클릭하면 다음 정보를 보여준다.

```text
OO님의 AI 카드
등급: SSS
점수: 96.4
현재 주간 랭킹: 7위
[나도 카드 만들기]
[앱에서 보기]
[리더보드 보기]
```

---

### 6.2 공유 이미지 규격

필요한 이미지 버전:

| 용도 | 규격 |
|---|---|
| 앱 내 카드 | 고해상도 PNG |
| 기본 공유 이미지 | 1200×630 또는 1080×1080 |
| 카카오톡 공유 | 카드 이미지 + 링크 |
| 인스타 스토리 | 1080×1920 |
| 인스타 피드 | 1080×1350 |
| 페이스북 공유 | OG 이미지 1200×630 |
| 다운로드 저장 | PNG/JPG |

이미지 생성 시 다음 버전을 자동 생성한다.

```text
card_original.png
card_square.png
card_story.png
card_feed.png
card_og.png
```

### 6.3 카카오톡 공유

카카오톡 공유는 “카카오톡 공유하기” SDK를 사용한다.

공유 템플릿 예시:

```text
[카드 이미지]

OO님의 AI 카드가 생성되었습니다!

등급: SSS
점수: 96.4
현재 주간 랭킹: 7위

버튼 1: 내 카드 만들기
버튼 2: 리더보드 보기
```

필수 구현:

```text
카카오톡 공유 SDK 연동
공유 버튼
공유용 이미지 URL
공유용 랜딩 URL
공유 성공/실패 콜백 로깅
가능하다면 공유 완료 웹훅 처리
```

주의:

- 공유 버튼 클릭만 이벤트로 보지 말고, 가능한 경우 공유 완료 기준으로 로그를 남긴다.
- 카카오톡 공유를 통한 직접 홍보 메시지 자동 발송은 하지 않는다.
- 사용자가 직접 공유 대상을 선택하게 한다.

### 6.4 인스타그램 공유

인스타그램은 자동 게시가 아니라, 사용자가 Instagram 앱으로 이동해 직접 올리는 구조로 구현한다.

버튼을 분리한다.

```text
인스타 스토리에 공유
인스타 피드에 공유
```

인스타 스토리 이미지 구성 예시:

```text
상단: 내 등급 SSS
중앙: 카드 이미지
하단: 현재 랭킹 7위 / 나도 만들기
```

주의:

- 인스타그램에 실제 게시 완료됐는지 앱이 항상 정확히 알 수 없을 수 있다.
- 따라서 상품 랭킹 산정에는 인스타 공유 횟수를 직접 반영하지 않는 것이 좋다.
- 인스타 공유는 바이럴 유입용으로 사용한다.
- 상품 랭킹은 앱 서버에 제출된 카드 점수 기준으로 운영한다.

### 7.4 추천 카드 레이아웃

공식 포켓몬 카드와 다른 배치를 사용한다.

추천 레이아웃:

```text

우측 상단: SCORE 또는 PWR
중앙: 사용자 사진 이미지 박스
좌측 세로바: 속성 아이콘
중하단: AI 분석 결과 요약

하단 2: 이벤트 랭킹 / 카드 ID
맨 아래: 서비스 브랜드 로고
```

주의:

- HP/속성/이름의 배치가 포켓몬 카드와 동일하지 않도록 한다.
- 약점/저항력/후퇴비용 같은 3단 하단 구조를 그대로 만들지 않는다.
- GX/EX처럼 보이는 큰 하단 스킬명 표현을 피한다.
- 자체 카드 세계관을 만든다.

## 8. IP 리스크 관리 체크리스트

## 21. 최종 성공 기준

MVP 성공 기준:

```text
사용자가 사진을 업로드하면 30초 이내 카드 생성 결과를 볼 수 있다.
카드에는 점수, 등급, 희귀도, 속성, PWR, 스킬이 자동 입력된다.
사용자는 점수와 등급을 수정할 수 없다.
카드는 리더보드에 등록된다.
사용자는 자신의 순위를 확인할 수 있다.
카카오톡, 인스타그램, 페이스북으로 공유할 수 있다.
공유 링크를 누르면 카드 상세 페이지가 열린다.
카드 상세 페이지에서 신규 사용자가 다시 카드 만들기로 진입할 수 있다.
관리자는 부적절 카드와 부정행위 카드를 숨김/삭제/랭킹 제외할 수 있다.
포켓몬 카드와 혼동될 정도의 UI, 명칭, 아이콘, 로고를 사용하지 않는다.
```

## 24. 마지막 요청

8. 공유 플로우 설계

```text
우선 코드를 바로 수정하지 말고, 현재 레포지토리 구조를 먼저 분석해줘.

중요 원칙:
- 점수/등급/능력치는 사용자가 수정할 수 없게 한다.
- 점수 계산과 리더보드는 서버 기준으로 한다.

- 트레이딩 카드 장르의 문법만 활용하고 자체 디자인/자체 세계관으로 구현한다.
```
